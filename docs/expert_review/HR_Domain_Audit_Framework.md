# HR 领域 AI 知识提取框架审核手册 (V1.0)

本项目旨在通过 **AI 互博 (AI Self-Play)** 的方式，逆向工程提取人力资源领域的专项技能知识点。为了确保 AI 提取的逻辑更接近人类顶级大师的水平，我们需要行业专家对以下三个核心底层框架进行审计和修正。

---

## 📋 模块一：HR 服务分类体系 (Taxonomy Audit)
**审核目的**：请确认服务分类是否专业、覆盖面是否齐全、术语是否准确。AI 将根据这套分类进行“导向型诊断”。

| 服务大类 | 子专项 / 节点 | 行业热词/关键词 | 专家修正意见 (留白) |
| :--- | :--- | :--- | :--- |
| **招聘与配置** | 高管猎头、蓝领RPO、校招代理、背调服务 | 快速到岗、胜任力匹配、人才入库、转化率 | |
| **薪酬福利** | 算薪外包、社保补缴方案、补充医疗、弹性福利 | 算薪合规、节税方案、个税递延、福利感知 | |
| **员工关系** | 劳动合同模板、补偿金预判、灵工转型、审计 | 仲裁预警、合规辞退、二阶段灵工、证据保存 | |
| **人才发展** | 素质模型、内训体系、任职资格评估 | 效能分析、文化落地、人才盘点、领导力 | |

## 📋 模块二：AI 互博行为逻辑 (Prompts Audit)
**审核目的**：确认“专家”的追问逻辑是否符合实务中的“挖需求”过程；确认“客户”表达是否真实。

### 1. 专家顾问 (Expert AI) 行为审计
*   **当前逻辑**：严禁直接推销。前 3 轮必须通过“漏斗提问法”确认客户的行业、规模和真实痛点。
*   **追问倾向**：优先区分客户是“合规性需求”还是“效能性需求”。
*   **专家纠偏提示**：*是否有某些关键问题是顶级顾问首轮必问的？对话节奏是否太快或太慢？*

### 2. 企业客户 (Novice AI) 行为审计
*   **当前逻辑**：禁止使用专业术语。用“怕麻烦、怕赔钱、人招不到”等口语表达。
*   **心理偏差**：会针对顾问表现出的专业度来调整“信任深度”。
*   **专家纠偏提示**：*真实的客户在咨询时是否有更典型的“防御心理”或“知识误区”？*

## 📋 模块三：问题场景生成框架 (Scenario Framework)
**审核目的**：确认该框架是否能“穷举”现实中的 HR 业务逻辑节点。

| 维度 | 生成逻辑 | 专家修正建议 |
| :--- | :--- | :--- |
| **身份变量** | 私企老板、初级HR、传统行业管理者、新经济HRD | 还需要增加哪类决策人？ |
| **行业变量** | 制造业(重离职率)、互联网(重期权/竞业)、服务业(重社保) | 行业痛点映射是否准确？ |
| **情绪变量** | 焦急(出事了)、保守(怕花钱)、专业但迷茫(新规后) | 情绪对对话走向的影响是否合理？ |
| **干扰因素** | 提供虚假预算、隐藏真实员工数、第三方服务对比 | 现实中还有哪些常见的沟通陷阱？ |

---

## 📬 专家反馈指引
请专家在上述表格的“专家修正意见”栏目中直接填写建议。重点关注：
1. **真实性**：AI 的话术是否“像个内行”？
2. **深度**：是否能挖掘出那些“水面下”的隐性需求？
3. **术语**：分类和关键词是否符合当前最新的行业监管及咨询习惯？
