# 🏦 保险密心 (Insurance Meseeing)

## 专为人力资源行业保险服务供应商设计的 AI 员工知识库提取工具

---

### 📋 项目概述

保险密心是"密心体系"在 **人力资源行业保险服务供应商** 领域的垂直应用。通过 AI 保险顾问与 AI 企业客户的多轮博弈对话，逆向工程提取保险顾问的专业诊断逻辑和需求识别技巧。

#### 核心价值
- 🎯 **精准需求诊断**：通过仿真对话，训练 AI 准确识别企业保险需求
- 📚 **专家知识沉淀**：将资深保险顾问的经验转化为可复用的知识库
- 🤖 **AI 员工培养**：为保险服务供应商打造专业的数字员工基础

### 🎯 服务分类体系 (Taxonomy)

| 大类 | 包含服务 |
|------|----------|
| **团体员工保险** | 团体医疗险、团体意外险、定期寿险、补充医疗、重疾团险 |

| **风险管理与理赔服务** | 风险评估、保单检查、批量理赔、续保优化、全案代理 |

### 🚀 快速开始

#### 1. 环境配置

```bash
# 复制环境变量模板
cp backend/.env.example backend/.env

# 编辑 .env，填入智谱 AI 密钥
OPENAI_API_KEY=你的密钥
OPENAI_API_BASE=https://open.bigmodel.cn/api/paas/v4/
```

#### 2. 启动服务

**🐳 方式一：Docker 模式（强烈推荐）**

✅ 优势：避免终端会话阻塞，环境隔离，一键启动

```bash
# 一键启动
bash docker_start.sh

# 或直接使用 Docker Compose
docker-compose up -d
```

> 💡 **为什么推荐 Docker？**
> - 解决 IDE Agent 终端卡顿问题
> - 服务在后台运行，不占用终端会话
> - 环境完全隔离，依赖统一管理
> 
> 详见：[Docker 使用指南](./DOCKER_GUIDE.md)

**⚙️ 方式二：本地运行**

```bash
# 一键启动（后台模式）
bash quick_start.sh

# 或使用原始脚本
bash start.sh
```

#### 3. 验证服务

访问浏览器：
- **前端应用**：http://localhost:3001
- **API 文档**：http://localhost:8001/docs

命令行验证：
```bash
# 健康检查
curl http://localhost:8001/health

# 获取服务分类
curl http://localhost:8001/api/taxonomy
```

#### 4. 停止服务

```bash
# Docker 模式
bash docker_stop.sh
# 或
docker-compose down

# 本地模式
bash stop_services.sh
```

### 📡 API 接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/start` | POST | 启动保险诊断仿真 |
| `/api/next` | POST | 执行仿真下一步 |
| `/api/etl/inbox` | GET | 获取待审核诊断记录 |
| `/api/ingest` | POST | 将记录入库知识库 |
| `/api/taxonomy` | GET | 获取服务分类体系 |
| `/api/coverage` | GET | 获取知识库覆盖率 |
| `/api/batch/start` | POST | 启动批量挖掘任务 |
| `/api/batch/status` | GET | 获取批量任务状态 |

### 🔧 与 HR 密心的区别

| 维度 | HR 密心 | 保险密心 |
|------|---------|----------|
| **目标用户** | HR 服务需求者 | 企业保险采购决策者 |
| **专家角色** | HR 服务顾问 | 保险需求诊断顾问 |
| **小白角色** | 普通员工/老板 | 企业管理者 (老板/HR/CFO) |
| **服务分类** | 招聘、薪酬、合规等 | 团险、雇责、弹福、高管险等 |
| **诊断逻辑** | 判断 HR 服务类型 | 判断保险产品需求 |
| **运行端口** | 8000/3000 | 8001/3001 |
